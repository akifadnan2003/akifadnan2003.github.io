Call Transcription & Analysis

Deepgram-Powered Speech-to-Text with RAG Knowledge Base Integration

Python Deepgram Flask Supabase SQLAlchemy RAG PII Masking
Back to Projects

Overview

A Flask-based service that transcribes customer support calls via the Deepgram speech-to-text API, enriches each transcript with product and knowledge-base context through a RAG pipeline, and stores structured call records in Supabase — with automatic PII masking applied before any data is written to disk or the database.

The system integrates a product catalogue and a support knowledge base as retrieval sources, allowing the analysis layer to surface relevant product information and support articles alongside each transcript — providing agents with instant context when reviewing a call.

Features

Deepgram STT Integration

Audio recordings are submitted to the Deepgram API which returns speaker-diarised, punctuated transcripts. The integration supports both file upload and streaming modes.

RAG-Enriched Transcripts

After transcription, the text is used as a query against the product catalogue and blog knowledge base. Relevant entries are attached to the transcript record, giving reviewers instant context.

PII Masking

Before any transcript is stored, a PII masking layer detects and replaces names, phone numbers, addresses, and order identifiers with placeholders — ensuring GDPR compliance.

Dual Storage Backend

Supabase serves as the primary cloud database via SQLAlchemy ORM. An SQLite database acts as a local fallback and vector index store, ensuring the service works offline for development.

REST API

All transcription, retrieval, and storage operations are exposed via a documented REST API, making the service composable with other systems (e.g. call recording platforms, CRM tools).

Web UI for Review

A Flask-served HTML interface lets support managers browse stored transcripts, search by keyword, and view the RAG-surfaced product context alongside each call record.
