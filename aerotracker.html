Overview

A full-stack delivery tracking platform that consolidates shipment data from DHL and UPS into a single real-time dashboard. Users can look up any order number and immediately see the current delivery status, carrier event history, and estimated delivery window — without switching between multiple carrier portals.

The backend is a Python FastAPI service deployed on Google Cloud Run, storing all tracking data in BigQuery. The frontend is built with Next.js 16 and React 19, deployed on Vercel, and communicates with the API through an authenticated proxy layer.

Features

Unified Multi-Carrier View

DHL and UPS use different status codes and event formats. The backend normalises both into a single 5-state model so the dashboard always shows consistent, human-readable status regardless of carrier.

Live Status Updates (SSE)

Server-Sent Events push tracking updates directly to the browser. Status changes appear instantly without polling or page reloads, giving a real-time feel to the dashboard.

Full Event History

Every carrier checkpoint event (departed facility, out for delivery, delivered, etc.) is stored in BigQuery and displayed as a chronological timeline on the shipment detail view.

Delay Detection

Shipments that exceed their expected delivery window are automatically flagged as delayed. The system compares current carrier status against the original ETA and surfaces overdue orders at the top of the dashboard.

Secure Authentication

JWT session cookies protect all API routes. Role-based access control ensures only authorised users can trigger re-scans or view sensitive order data.

Cloud-Native Deployment

The FastAPI backend runs as a stateless container on Google Cloud Run, scaling to zero when idle and scaling out automatically under load. The Next.js frontend is deployed on Vercel with edge caching.
