Overview

A production-grade full-stack shipment tracking platform that consolidates DHL and UPS carrier data into a single real-time dashboard. Customer service teams can search any order number, see live shipment status, and take bulk administrative actions — all from a unified web interface.

The backend runs on Google Cloud Run with a nightly Cloud Scheduler job that re-checks all non-delivered orders Monday through Friday at 21:00 (TRT), processing 1,200+ orders per run. All data is stored in BigQuery for historical analysis and monthly KPI reporting.

Features

Real-Time SSE Progress

Server-Sent Events stream live scan progress to the dashboard during mass re-scan operations. A real-time progress bar shows exactly how many orders have been checked, with no page refresh needed.

Multi-Carrier Support

DHL Tracking API and UPS OAuth 2.0 integration normalise carrier-specific status codes into a unified 5-state classification system, so agents see consistent status regardless of carrier.

Admin Bulk Operations

Admins can re-scan individual orders, trigger mass scans across all active shipments, or force a full sync from the carrier APIs. All operations are protected by JWT RBAC.

Nightly Automation

Cloud Scheduler triggers the nightly batch job (Mon–Fri 21:00 TRT) which re-checks every non-delivered order against live carrier APIs and updates BigQuery — keeping data fresh without manual intervention.

Delay Alerts via Helpdesk

When the tracking engine detects a shipment that has exceeded its SLA window, it automatically creates a support ticket with full shipment context attached for the operations team to action.

Monthly KPI Reports

A dedicated reports page aggregates BigQuery data into monthly KPIs: delivery rate, average transit time, delay rate by carrier, and return rate. Exportable as CSV for external reporting.
