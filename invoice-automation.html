Invoice Automation System

LLM-Assisted Billing Validation, Comment Auditing & Invoice Generation

Python Ollama LLM CSV / Excel Data Processing
Back to Projects

Overview

A Python-based automation system that handles the full invoice preparation pipeline: reading raw order data, validating billing comments against a structured rule set, detecting discrepancies, and generating final billing documents ready for issue.

The system uses a locally-hosted LLM (via Ollama) to interpret free-text order comments — a task previously done manually — and maps each comment to the corresponding billing rule. This removes ambiguity from the validation step and produces an auditable paper trail of every decision made.

Features

LLM Comment Parsing

Free-text billing comments from order exports are sent to a locally-running Ollama model which classifies each comment against the defined billing rules — eliminating manual interpretation.

Rules Engine

A structured billing rules directory encodes all valid billing scenarios. The calculator validates each order line against the rules and flags mismatches for human review.

Automated Document Generation

Once validated, the billing document builder assembles final invoices from the processed data, formatting all required fields and totals without manual data entry.

Audit Trail

Every validation decision is logged to an Excel audit report, providing a full audit trail that links each order line to the rule used and the LLM's interpretation of the comment.

PII Protection

A dedicated PII processor strips personal identifiers from order data before processing, ensuring customer data is handled in compliance with data protection requirements.

API Audit Layer

An API audit module tracks all external data fetches and validates returned data shape against expected schemas, preventing silent failures from upstream data changes.
