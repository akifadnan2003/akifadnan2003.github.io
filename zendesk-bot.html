Overview

An end-to-end AI support automation platform that eliminates manual ticket handling by automatically classifying, routing, and resolving incoming customer support tickets. The system integrates directly with a helpdesk platform via its REST API, enabling zero-touch resolution for a wide range of request types.

At its core, the bot uses a Retrieval-Augmented Generation (RAG) pipeline — querying a vector knowledge base of historical tickets and documentation — to generate contextually accurate responses. Dual LLM support (Google Gemini and Claude) allows for model failover and A/B comparison. All ticket data and metrics flow into BigQuery for long-term analytics.

Features

Semantic Routing

Incoming ticket text is embedded and matched against intent clusters using cosine similarity, directing each ticket to the most appropriate specialised handler without rule-based keyword matching.

RAG Knowledge Base

A local vector store built from historical tickets and documentation chunks enables grounded, citation-backed responses. New tickets continuously enrich the store.

RMA & Return Handling

Automated RMA workflow checks eligibility against return policy rules, drafts approval or rejection replies, and updates the ticket status — no human intervention needed for standard cases.

Escalation Engine

Configurable escalation rules detect high-priority signals (VIP customers, repeated contacts, negative sentiment) and route tickets to the correct support group with full context attached.

BigQuery Analytics

Every resolution event, handler decision, and LLM call is logged to BigQuery, enabling dashboards on resolution rates, average handle time, and escalation frequency.

Docker Deployment

Containerised FastAPI service with environment-based configuration, making it deployable to any cloud provider with a single docker-compose command.
